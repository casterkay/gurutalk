## 1. 对本项目的根本认识（不可违背的事实）

### 1.1 产品分工（最重要）

- `Bibliotalk` 是云端 API + 引用短链服务：提供 **人物目录**、**Persona Profile**、**检索结果**、**引用短链页 `/q/:quote_id`**、**稳定分享页 `/pub/:share_id`**。
- `Bibliotalk` **不生成最终回答、不调用 LLM**。任何回答与风格控制由客户端/个人 Agent 完成。

### 1.2 MVP 的第一性目标：端到端闭环（E2E）

最小闭环必须尽快跑通（优先于“优雅架构”）：

1) 离线摄入写入记忆库（按人物 `slug` 分容器）  
2) `POST /v1/query` 检索并返回 `results[]`（`kind="chunk"` + 可选 `kind="memory"`）  
3) `chunk` 生成 `quote_id` 并写入 KV（`q:{quote_id}`，**30 天 TTL**）  
4) `/q/{quote_id}` 引用页可打开（含 Open Graph meta）  
5) `/q/{quote_id}` 触发分享生成 `share_id`，写 KV（`pub:{share_id}`，**永久快照**）  
6) `/pub/{share_id}` 分享页可长期访问并带 CTA

> 一切开发顺序以 `bibliotalk/PLAN.md` 的 Sprint 1 为准，任务以 `bibliotalk/TASKS.md` 为执行入口。

## 3. 开发规程（优秀 AI 智能体的基本素养）

### 3.1 先对齐，再动手

- 在写代码之前，先明确你正在实现 `bibliotalk/TASKS.md` 里的哪张卡（例如 BT-031）。
- 不要“顺手”扩 scope；任何超出当前卡片验收条件的改动都应拆到新卡片。

### 3.2 变更要可验证、可回滚

- 每个任务必须能用 **最小可复现步骤**验证（优先 curl / 浏览器 / 简单脚本）。
- 不引入一次性的大重构；优先小步提交式改动（本仓库无需你自动 `git commit`，但变更应保持可分割）。

### 3.3 代码与架构整洁度（必须重视）

本仓库强调：**简洁、可靠、小而精、模块化、良好封装、避免重复、主动清理**。但“主动清理”不等于无边界重构：应当在不扩大当前任务范围的前提下，让你触碰的代码更清爽、更一致、更可维护。

硬性要求：
- **优先最小可用的清晰实现**：能直观读懂、能快速验证、能安全演进。
- **模块化与分层**：路由（HTTP）/ 业务（服务）/ 存储（KV）/ 上游适配（记忆库）/ 渲染（HTML）/ 工具（id、校验）分离，避免“巨型 handler 文件”。
- **封装边界清晰**：不要在路由层散落 KV key 拼接、hash 规则、HTML 拼接细节；这些应集中在模块里统一维护。
- **避免重复**：重复的字段映射、错误响应、KV 读写、参数校验要抽出可复用的函数/类型；不要 copy-paste 逻辑。
- **可靠性优先**：确定性 `quote_id`、显式错误语义、输入校验、空结果可接受；禁止“看似成功但实际丢数据”的静默失败。
- **渐进清理（Boy Scout Rule）**：改动某文件时，顺手清理该文件内的明显技术债（命名、重复、死代码、错误处理不一致），但不要跨模块大迁移。

建议做法（不强制，但强烈推荐）：
- 把所有环境变量/配置读取集中在单一模块，避免到处 `env.X`。
- 统一定义请求/响应 DTO 类型与校验（至少保证字段名与 PRD 一致）。
- 统一错误返回格式与错误码映射（API 与页面都要一致表述“引用不存在/已过期”“分享不存在”等）。

### 3.3 错误语义必须清晰

至少要清楚区分并在接口/页面上返回一致语义：
- 人物不存在 / 不可用
- 检索结果为空
- 引用不存在或已过期（`/q/{quote_id}`）
- 分享链接不存在（`/pub/{share_id}`）

### 3.4 安全与合规（默认从严）

- **不要提交任何密钥/Token**（`.env`、KV binding、上游 API key、OAuth token 等）。
- 引用展示遵循“短摘 + 明确来源”原则；避免输出大段受版权保护原文。
- 对外页面（`/q`、`/pub`）不要埋入与需求无关的追踪代码或第三方脚本，除非明确任务要求。

## 4. 实现约束（对齐 PRD 的关键点）

### 4.1 `POST /v1/query` 结果约束

- `results[]` 允许混合两类：
  - `kind="chunk"`：可展示原文片段，**必须有** `quote_id`
  - `kind="memory"`：归纳/记忆条目，**不得有** `quote_id`

### 4.2 引用生命周期约束

- 临时引用：KV `q:{quote_id}`，**30 天 TTL**；过期后 `/q/{quote_id}` 明确提示“引用不存在”。
- 稳定分享：KV `pub:{share_id}`，**不设置 TTL**；用于保证临时引用过期后仍可访问。

### 4.3 Profile 版本约束

- `profile_version` 采用“UTC 日期 + 当日递增后缀”（例如 `2026-04-04.1`）。
- 客户端只需按字典序比较版本字符串即可。

- **只做能演示的垂直切片**：先满足 Sprint 1 的 smoke checklist，再讨论抽象与复用。

  2) 记忆库检索适配
  3) quote_id 与 KV 写入

  5) share 生成与 `/pub` 页面
  6) 鉴权/限流/可观测（硬化）
